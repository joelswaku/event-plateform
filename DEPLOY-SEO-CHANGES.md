# How to Deploy SEO Changes to Production

## 🎯 Quick Answer

Yes, you just need to **push to git** and your GitHub Actions will automatically deploy to production!

## 📋 Step-by-Step Deployment

### Step 1: Commit & Push Changes

```bash
# Check what files were changed
git status

# Add all SEO-related files
git add web/src/app/sitemap.ts
git add web/public/robots.txt
git add web/src/app/layout.js
git add web/src/app/e/[slug]/page.js
git add api/services/events.service.js
git add api/controllers/events.controller.js
git add api/routes/events.routes.js
git add web/SEO-SETUP.md
git add web/SEO-DEPLOYMENT-CHECKLIST.md
git add web/QUICK-START-SEO.md
git add SEO-IMPLEMENTATION-SUMMARY.md

# Commit the changes
git commit -m "Add comprehensive SEO sitemap implementation

- Add dynamic sitemap.ts with hourly revalidation
- Add robots.txt for search engine directives
- Enhance metadata with Open Graph and Twitter cards
- Add JSON-LD structured data for events
- Create API endpoint /events/public-sitemap
- Add complete SEO documentation"

# Push to main branch (triggers production deployment)
git push origin main
```

### Step 2: Monitor Deployment

1. **Go to GitHub Actions**: https://github.com/YOUR_USERNAME/event-plateform/actions

2. **Watch the workflow**:
   - You'll see "Deploy to Production" workflow running
   - It requires **MANUAL APPROVAL** in the `production` environment
   - You'll get a notification to approve

3. **Approve the deployment**:
   - Click on the workflow run
   - Click "Review deployments"
   - Select "production"
   - Click "Approve and deploy"

4. **Wait for completion** (~5-10 minutes):
   - ✅ Build & Push Images
   - ✅ Run Database Migrations (if needed)
   - ✅ Deploy API Service
   - ✅ Deploy Web Service
   - ✅ Deploy Vendors Service
   - ✅ Health Checks

### Step 3: Verify Deployment

Once deployment is complete:

```bash
# 1. Check sitemap
curl https://liteevent.com/sitemap.xml

# 2. Check robots.txt
curl https://liteevent.com/robots.txt

# 3. Check API endpoint
curl https://api.liteevent.com/api/events/public-sitemap

# 4. Open in browser
# https://liteevent.com/sitemap.xml
# https://liteevent.com/robots.txt
```

Expected results:
- ✅ Sitemap shows XML with URLs
- ✅ Robots.txt shows crawling rules
- ✅ API returns JSON with events

### Step 4: Submit to Google (After Verification)

Follow the [QUICK-START-SEO.md](web/QUICK-START-SEO.md) guide to submit your sitemap to Google Search Console.

## 📊 What Gets Deployed

### Backend API Changes
- **New endpoint**: `GET /api/events/public-sitemap`
- Returns all PUBLISHED + PUBLIC events
- Used by sitemap for dynamic URL generation

### Frontend Changes
- **New route**: `/sitemap.xml` (auto-generated by Next.js)
- **New file**: `/robots.txt` (in public folder)
- **Enhanced metadata**: Better SEO tags on all pages
- **Structured data**: JSON-LD schema on event pages

## ⚠️ Important Notes

### Database Impact
- No new tables or migrations required
- Only reads existing `events` table
- Query filters: `status='PUBLISHED' AND visibility='PUBLIC'`

### Caching
- Sitemap regenerates every hour (ISR)
- First request after deployment will be slow (~2-3 seconds)
- Subsequent requests are instant (cached)

### No Downtime
- Zero downtime deployment
- API and Web deploy sequentially
- Health checks ensure stability

## 🐛 Troubleshooting

### If deployment fails:

1. **Check GitHub Actions logs**:
   - Go to Actions tab in GitHub
   - Click on the failed workflow
   - Review error messages

2. **Common issues**:
   - Build errors: Check Docker build logs
   - API health check fails: Check ECS logs in AWS Console
   - Timeout: Services taking too long to stabilize

3. **Rollback if needed**:
   ```bash
   # In AWS Console:
   # ECS → Clusters → liteevent-production-cluster
   # Select service → Update → Previous task definition
   ```

### If sitemap returns 404:

1. **Check Next.js build logs** in GitHub Actions
2. **Verify file location**: `web/src/app/sitemap.ts` (not `pages/`)
3. **Check build output** for route generation
4. **Try manual rebuild**:
   ```bash
   cd web
   npm run build
   # Check for /sitemap.xml in build output
   ```

### If sitemap is empty:

1. **Check API endpoint**:
   ```bash
   curl https://api.liteevent.com/api/events/public-sitemap
   ```
2. **Verify database has published events**:
   ```sql
   SELECT COUNT(*) FROM events 
   WHERE status='PUBLISHED' AND visibility='PUBLIC' AND deleted_at IS NULL;
   ```
3. **Check environment variables**:
   - `NEXT_PUBLIC_API_URL` must be set correctly
